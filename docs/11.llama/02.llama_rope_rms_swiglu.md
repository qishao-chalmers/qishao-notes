## Part 1: Rotary Position Embedding (RoPE)

### 1. Theoretical Matrix vs. Implementation
* **Logical Dimension:** The rotary matrix $\mathcal{R}$ is effectively $d \times d$, where $d$ is the head dimension.
* **Actual Structure:** It is a **sparse, block-diagonal matrix** composed of $2 \times 2$ rotation blocks along the diagonal.
* **Function:** It rotates pairs of elements $(x_{2i}, x_{2i+1})$ by an angle $m\theta_i$, where $m$ is the token position and $\theta_i$ is the frequency for that pair.

### 2. The "Rotate Half" Trick
To optimize for GPUs (avoiding scattered memory access and ensuring contiguous memory blocks), we do not pair immediate neighbors $(x_0, x_1)$. Instead, we split the head dimension into two distinct halves.

* **Pairing Strategy:** Index $i$ is paired with Index $i + d/2$.
* **Vector Split:** The input vector $x$ is split into `left` ($x_1$) and `right` ($x_2$).
* **Transformation:**
`rotate_half(x) = [-x_2, x_1]`
  
    *(Mathematically equivalent to concatenating negated right half and original left half).*
* **The RoPE Formula:**

$$
\mathrm{RoPE}(x, m) = x \odot \cos(m\Theta) + \mathrm{rotate}_{\mathrm{half}}(x) \odot \sin(m\Theta)
$$


### 3. PyTorch Implementation
Reference implementation using the cache optimization:

```python
import torch

def rotate_half(x):
    """
    Splits the last dimension into two halves.
    Transforms [x1, x2] -> [-x2, x1]
    """
    x1 = x[..., : x.shape[-1] // 2]
    x2 = x[..., x.shape[-1] // 2 :]
    return torch.cat((-x2, x1), dim=-1)

def apply_rotary_pos_emb(q, k, cos, sin):
    """
    Args:
        q, k: [Batch, Heads, SeqLen, HeadDim]
        cos, sin: [1, 1, SeqLen, HeadDim] (Broadcastable)
    """
    q_embed = (q * cos) + (rotate_half(q) * sin)
    k_embed = (k * cos) + (rotate_half(k) * sin)
    return q_embed, k_embed
```

To implement LLaMA 2.0 from scratch in CUDA, you need to implement or wrap the following operations:

| Component | Kernel Name | Complexity | Key Implementation Details |
| :--- | :--- | :--- | :--- |
| **Normalization** | `RMSNorm` | Medium | Fused reduction + normalization. Critical for numerical stability. |
| **Activation** | `SwiGLU` | Easy | Element-wise. Fuses SiLU activation and Gating multiplication. |
| **Linear Layers** | `MatMul` (GEMM) | Hard | Use **cuBLAS**. Note: LLaMA linear layers have **no bias**. |
| **Attention** | `KV_Cache_Update` | Easy | Memory copy kernel to store current token's K/V into the cache buffer. |

| **Attention** | `Softmax` | Medium | Requires stable reduction (subtract max value before exp). |

## Part 3: Deep Dive - RMSNorm Kernel

### 1. Mechanism
RMSNorm normalizes the input vector by its Root Mean Square (no mean subtraction like LayerNorm).
$$\bar{x}_i = \frac{x_i}{\sqrt{\frac{1}{n}\sum x^2 + \epsilon}} \cdot w_i$$

### 2. Implementation Strategy
* **Grid Structure:** **1 Block = 1 Row (Token).**
    * Do **not** split a single `d_model` row across multiple blocks. The synchronization cost exceeds the compute benefit.
* **Block Size:** 256 threads (Standard sweet spot).
* **Looping (Striding):** If $d_{model} > (\text{BlockSize} \times \text{VecWidth})$, the block must loop over the data.
    * Stride = `blockDim.x * ElementsPerThread`.
    * Threads process indices: $tid, tid+stride, tid+2*stride...$

### 3. Optimization: Warp Shuffles
For the reduction (sum of squares), use **Warp Shuffles** (`__shfl_down_sync`) instead of Shared Memory where possible.
1.  Threads sum their local values.
2.  Warps reduce using shuffles (register-to-register).
3.  First thread of each warp writes to Shared Memory.
4.  First warp reduces the Shared Memory values.

**Critical Note:** Accumulate sums in **FP32** (`float`) even if the input is FP16 (`__half`) to avoid overflow and precision loss.


```cpp 
#include <cuda_runtime.h>

#define WARP_SIZE 32

// 1. The Warp Shuffle Helper
__device__ inline float warpReduceSum(float val) {
    for (int offset = WARP_SIZE / 2; offset > 0; offset /= 2) {
        val += __shfl_down_sync(0xffffffff, val, offset);
    }
    return val;
}

// 2. The Block Reduce Helper
// Requires shared memory sized [BlockSize / 32] (max 32 elements for block=1024)
__device__ inline float blockReduceSum(float val) {
    
    // A. Every warp performs a reduction
    float warp_sum = warpReduceSum(val);

    // Get thread info
    int lane = threadIdx.x % WARP_SIZE; // 0..31
    int warp_id = threadIdx.x / WARP_SIZE;

    // Shared memory to store partial sums from each warp
    // Assuming max block size of 1024, we need 32 float slots.
    __shared__ float shared_warp_sums[32]; 

    // B. The "Leader" of each warp writes to shared memory
    if (lane == 0) {
        shared_warp_sums[warp_id] = warp_sum;
    }
    
    // Wait for all warps to write
    __syncthreads();

    // C. The First Warp reads the partial sums and reduces them
    // Note: We only need the first warp (threads 0-31) to do this work.
    val = (threadIdx.x < blockDim.x / WARP_SIZE) ? shared_warp_sums[lane] : 0.0f;
    
    if (warp_id == 0) {
        val = warpReduceSum(val);
    }

    // Now Thread 0 holds the TOTAL sum of the entire block.
    return val;
}

__global__ void rms_norm_kernel_optimized(
    const float* __restrict__ input,
    const float* __restrict__ weight,
    float* __restrict__ output,
    int hidden_dim,
    float epsilon) 
{
    int row_idx = blockIdx.x;
    int tid = threadIdx.x;

    const float* row_input = input + row_idx * hidden_dim;
    float* row_output = output + row_idx * hidden_dim;

    // 1. Thread Local Accumulation
    // Each thread sums up multiple elements (strided loop)
    float local_sum_sq = 0.0f;
    for (int i = tid; i < hidden_dim; i += blockDim.x) {
        float val = row_input[i];
        local_sum_sq += val * val;
    }

    // 2. Block Reduction (The Warp Shuffle Magic)
    // All threads call this, but the result ends up in Thread 0's register
    float total_sum_sq = blockReduceSum(local_sum_sq);

    // 3. Broadcast Result
    // Thread 0 calculates the scalar, writes to shared mem so everyone can read it
    __shared__ float inv_rms;
    if (tid == 0) {
        inv_rms = rsqrtf(total_sum_sq / hidden_dim + epsilon);
    }
    __syncthreads(); // Wait for Thread 0 to write

    // 4. Apply
    float rms_scalar = inv_rms; // Read from shared mem to register
    for (int i = tid; i < hidden_dim; i += blockDim.x) {
        row_output[i] = row_input[i] * rms_scalar * weight[i];
    }
}
```


```cpp
#include <cuda_fp16.h>

#define BLOCK_SIZE 256
#define VEC_SIZE 8 // 8 elements per thread for FP16 (128 bits)

// Helper: Pack 8 halves into struct for cleaner code
struct __align__(16) half8 {
    __half2 v[4];
};

__global__ void rms_norm_kernel_loop(
    const __half* __restrict__ input,
    const __half* __restrict__ weight,
    __half* __restrict__ output,
    int d_model,
    float epsilon) 
{
    // 1. One Block per Row (Token)
    int row_idx = blockIdx.x;
    
    // Pointers to the start of this specific row
    const __half* row_input = input + (row_idx * d_model);
    __half* row_output = output + (row_idx * d_model);
    const __half* row_weight = weight; // Weights are shared across all rows

    // 2. Accumulation Loop
    // Initialize accumulation in FP32
    float thread_sum_sq = 0.0f;

    // "tid" is the thread index (0..255)
    // "stride" is how many elements the WHOLE block processes in one step (256 * 8 = 2048)
    int stride = blockDim.x * VEC_SIZE;

    // Loop until we cover the whole d_model
    for (int i = threadIdx.x * VEC_SIZE; i < d_model; i += stride) {
        
        // Load 8 elements (128 bits)
        // Note: You need boundary checks if d_model is not multiple of 8
        // For LLaMA, d_model is usually aligned, but be safe.
        
        half8 data_vec = *reinterpret_cast<const half8*>(&row_input[i]);
        
        // Sum squares for these 8 elements
        // Unrolling this loop manually or trusting compiler
        #pragma unroll
        for (int k = 0; k < 4; k++) {
            __half2 v2 = data_vec.v[k];
            // Convert to float, square, accumulate
            float2 f2 = __half22float2(v2);
            thread_sum_sq += f2.x * f2.x;
            thread_sum_sq += f2.y * f2.y;
        }
    }

    // 3. Block Reduction (Sum across threads)
    // Use the Warp Shuffle logic we discussed earlier
    float total_sum_sq = blockReduceSum(thread_sum_sq);

    // 4. Calculate RMS Scalar (Only Thread 0)
    __shared__ float inv_rms;
    if (threadIdx.x == 0) {
        inv_rms = rsqrtf(total_sum_sq / d_model + epsilon);
    }
    __syncthreads();

    // 5. Application Loop (Re-read and Apply)
    // We must loop again to write the output
    float scalar = inv_rms;

    for (int i = threadIdx.x * VEC_SIZE; i < d_model; i += stride) {
        // Re-load input (L1 cache usually makes this cheap)
        half8 in_vec = *reinterpret_cast<const half8*>(&row_input[i]);
        half8 w_vec  = *reinterpret_cast<const half8*>(&row_weight[i]);
        half8 out_vec;

        #pragma unroll
        for (int k = 0; k < 4; k++) {
            __half2 x = in_vec.v[k];
            __half2 w = w_vec.v[k];
            
            // output = x * w * scalar
            // Using __hmul2 for vector multiplication if possible, or convert to float
            float2 x_f2 = __half22float2(x);
            float2 w_f2 = __half22float2(w);
            
            float2 res_f2;
            res_f2.x = x_f2.x * w_f2.x * scalar;
            res_f2.y = x_f2.y * w_f2.y * scalar;
            
            out_vec.v[k] = __float22half2_rn(res_f2);
        }

        // Store 8 elements
        *reinterpret_cast<half8*>(&row_output[i]) = out_vec;
    }
}
```

## Part 4: Deep Dive - SwiGLU Kernel

### 1. Mechanism
SwiGLU replaces standard activation functions. It requires 3 linear projections, but 2 are usually fused.
1.  **Fused MatMul:** Produces a large tensor of size $[Batch, Seq, 2 \times H_{inter}]$.
2.  **Kernel Split:** Logically separates the first half (Gate) and second half (Value).
3.  **Activation:** $\text{Output} = \text{SiLU}(\text{Gate}) \odot \text{Value}$.

### 2. Dimensions & Data Flow
* **Input:** Flattened array of size $N = Batch \times Seq \times (2 \cdot H_{inter})$.
* **Logic:**
    * Thread $i$ reads Gate at `input[i]`.
    * Thread $i$ reads Value at `input[i + H_inter]`.
    * *Note: Ensure input layout is contiguous [Gate... | Value...], not interleaved.*
* **Output:** Flattened array of size $Batch \times Seq \times H_{inter}$.

### 3. Implementation Checklist
* **Pointer Arithmetic:** Use the offset `H_inter` to jump between Gate and Value parts.
* **Precision:** Compute $e^{-x}$ (for Sigmoid/SiLU) in **FP32**. FP16 has insufficient range for exponentials.
* **Grid:** Use a **1D Flattened Grid**. Treat the tensor as a 1D array; ignore Batch/Seq dimensions inside the kernel.


```cpp
#include <cuda_runtime.h>
#include <iostream>
#include <cmath>

// --- Device Helpers ---

__device__ __forceinline__ float sigmoid_opt(float x) {
    // Standard implementation. 
    // For extreme speed (approximate), use __fdividef(1.0f, 1.0f + __expf(-x));
    return 1.0f / (1.0f + expf(-x));
}

__device__ __forceinline__ float silu_opt(float x) {
    return x * sigmoid_opt(x);
}

// --- Optimized Kernel ---

/**
 * @brief Optimized SwiGLU Kernel
 * * Optimization Techniques:
 * 1. Vectorization: Uses float4 to load/store 128 bits (16 bytes) per instruction.
 * 2. Reduced Indexing: Calculates overhead once per 4 elements.
 * * Constraints:
 * - 'size' (total elements) must be divisible by 4.
 * - Input pointers must be aligned to 16 bytes (standard in cudaMalloc).
 */
__global__ void swiglu_kernel_vectorized(
    const float* __restrict__ gate_input,
    const float* __restrict__ val_input,
    float* __restrict__ output,
    int total_elements) 
{
    // The Global Index here represents the "Vector Index", not scalar index.
    // If blockDim is 256, thread 0 handles elements 0,1,2,3. Thread 1 handles 4,5,6,7.
    int idx = blockIdx.x * blockDim.x + threadIdx.x;

    // We process 'float4' chunks, so we check against (size / 4)
    if (idx < (total_elements / 4)) {
        
        // 1. Vectorized Load (LDG.128)
        // Reinterpret the raw float pointer as a float4 pointer
        float4 g_vec = reinterpret_cast<const float4*>(gate_input)[idx];
        float4 v_vec = reinterpret_cast<const float4*>(val_input)[idx];
        float4 out_vec;

        // 2. Compute (Unrolled manually)
        out_vec.x = silu_opt(g_vec.x) * v_vec.x;
        out_vec.y = silu_opt(g_vec.y) * v_vec.y;
        out_vec.z = silu_opt(g_vec.z) * v_vec.z;
        out_vec.w = silu_opt(g_vec.w) * v_vec.w;

        // 3. Vectorized Store (STG.128)
        reinterpret_cast<float4*>(output)[idx] = out_vec;
    }
}

// --- Host Wrapper ---

void launch_swiglu_optimized(const float* d_gate, const float* d_val, float* d_out, int N) {
    // Constraint Check: N must be divisible by 4 for this specific kernel
    if (N % 4 != 0) {
        std::cerr << "Error: Size N must be divisible by 4 for vectorized kernel." << std::endl;
        return;
    }

    int blockSize = 256;
    
    // Grid Calculation:
    // We are launching threads to cover N/4 vectors.
    int num_vectors = N / 4;
    int gridSize = (num_vectors + blockSize - 1) / blockSize;

    swiglu_kernel_vectorized<<<gridSize, blockSize>>>(d_gate, d_val, d_out, N);
}

// --- Main Verification ---

int main() {
    int N = 1024 * 1024; // 1 Million elements
    size_t bytes = N * sizeof(float);

    float *h_gate = new float[N];
    float *h_val  = new float[N];
    float *h_out  = new float[N];

    // Init data
    for(int i=0; i<N; i++) { h_gate[i] = 1.0f; h_val[i] = 2.0f; }

    float *d_gate, *d_val, *d_out;
    cudaMalloc(&d_gate, bytes);
    cudaMalloc(&d_val, bytes);
    cudaMalloc(&d_out, bytes);

    cudaMemcpy(d_gate, h_gate, bytes, cudaMemcpyHostToDevice);
    cudaMemcpy(d_val, h_val, bytes, cudaMemcpyHostToDevice);

    std::cout << "Launching Optimized SwiGLU..." << std::endl;
    launch_swiglu_optimized(d_gate, d_val, d_out, N);
    
    cudaMemcpy(h_out, d_out, bytes, cudaMemcpyDeviceToHost);

    // Verify
    std::cout << "Verified Index 0: " << h_out[0] << " (Expected ~1.462)" << std::endl;
    std::cout << "Verified Index 3: " << h_out[3] << " (Expected ~1.462)" << std::endl;

    cudaFree(d_gate); cudaFree(d_val); cudaFree(d_out);
    delete[] h_gate; delete[] h_val; delete[] h_out;

    return 0;
}
```

## Part 5: CUDA Optimization & Best Practices

### 1. Vectorization (`float4` / `long4`)
* **Concept:** GPU memory controllers are most efficient when loading **128 bits (16 bytes)** at a time.
* **Instruction:** This maps to a single `LDG.128` instruction, reducing instruction fetch/decode overhead by 75% compared to scalar loads.
* **Elements per Thread:**
    * **FP32 (`float`):** 1 `float4` = **4 elements**.
    * **FP16 (`half`):** 1 `float4` = **8 elements** (packed).

### 2. Handling FP16 in CUDA
* **Loading:** Cast input pointers from `half*` to `float4*` to perform the 128-bit load.
* **Type Punning:** Reinterpret the loaded `float4` variable as an array of `__half2` (SIMD pairs).
* **Code Pattern:**
    ```cpp
    // 1. Load 128 bits (16 bytes)
    float4 raw_data = ((float4*)input_ptr)[idx]; 
    
    // 2. Re-interpret bits as 4 pairs of half2
    half2* packed_pairs = (half2*)&raw_data;
    
    // 3. Process 8 elements
    for(int i=0; i<4; i++) {
        half2 val = packed_pairs[i];
        // ... math ...
    }
    ```

### 3. Grid & Block Sizing Formula
For element-wise kernels (like SwiGLU):
* **Block Size:** 256.
* **Formula:**
    $$\text{GridSize} = \left\lceil \frac{\text{Total Elements}}{\text{BlockSize} \times \text{ElementsPerThread}} \right\rceil$$
* *Example for FP16:* `(Total_Elements + 2047) / 2048`.

### 4. Memory Layout & Limits
* **Alignment:** Ensure hidden dimensions (e.g., $H_{inter}$) are divisible by the vector width (8 for FP16) to prevent misalignment.
* **Grid Limit:** The maximum `gridDim.x` is $2^{31}-1$ (~2 Billion). For current LLM tensor sizes, a 1D grid is safe. If exceeding this, use a 2D grid.
