# 1. [2025 New] Can LLMs Maintain Fundamental Abilities under KV Cache Compression? 👍

This is a good paper....but no source code

This paper investigates the impact of Key-Value (KV) cache compression on the fundamental capabilities of Large Language Models (LLMs).

The authors conduct a comprehensive empirical study to evaluate how different KV cache compression methods affect various tasks, including world knowledge, common-sense reasoning, arithmetic reasoning, code generation, safety, and long-context understanding and generation.

The study reveals that KV cache compression methods exhibit task-specific performance degradation, with arithmetic reasoning tasks being particularly sensitive to aggressive compression.

The key insight is that the prefill phase KV cache, which contains crucial prompt information, should be compressed once and remain fixed, while the decoding phase KV cache can be dynamically compressed with different strategies. Given a prefill compression ratio rp, we prioritize shots with higher scores while ensuring the total number of preserved tokens does not exceed the KV cache limit. Specifically, shots are ranked by their scores and selected in descending order until reaching the compression budget rp × |KVprefill|. This shot-level selection strategy helps maintain the semantic coherence of important examples while adhering to memory constraints.

# Key Findings

  • Task-Specific Performance Degradation:

    Different tasks show varying levels of sensitivity to KV cache compression.

    Arithmetic reasoning tasks are the most sensitive, with performance drops ranging from 17.4% to 43.3% under aggressive compression.

    In contrast, tasks like world knowledge and common-sense reasoning are more resilient.

  • Robustness of Multi-Step Reasoning Models:

    Multi-step reasoning LLMs, such as the DeepSeek R1 Distill model, exhibit more robust compression tolerance compared to instruction-tuned models.

    The DeepSeek R1 Distill model shows only 9.67%-25.53% performance degradation under compression.

  • Impact of Prompt Length:

    Shorter prompts are more vulnerable to compression effects.

    Performance degradation is more pronounced in one-shot and two-shot scenarios compared to prompts with more examples.

  • Chunk-Level Compression:

    Chunk-level compression strategies are more effective for complex long-context arithmetic reasoning tasks.

    These strategies help maintain semantic coherence and improve performance under aggressive compression.

  • Long-Context Generation Sensitivity:

    Long-context generation tasks are particularly sensitive to compression, with significant performance degradation observed at low compression ratios.

# Proposed Method

ShotKV: The authors propose ShotKV, a novel compression approach that distinctly handles the prefill and decoding phases while maintaining shot-level semantic coherence.

ShotKV achieves 9%-18% performance improvements on long-context generation tasks under aggressive compression ratios.

# Insights:

  • Attention Patterns:

    The study highlights that attention patterns vary significantly across different tasks.

    Arithmetic reasoning tasks display increased attention sparsity, while safety tasks show concentrated attention on system prompts. These patterns provide insights into how compression affects various tasks.

  • Compression Trade-offs:

    The findings suggest a trade-off between memory efficiency and task performance.

    While KV cache compression reduces memory usage, it can significantly impact model performance, especially in tasks requiring complex reasoning or extensive knowledge retrieval.

  • Model Design Implications: The higher sensitivity of instruction-tuned models to compression raises questions about the relationship between model training objectives and compression robustness.

    Future research could explore training techniques that optimize for compression resilience while maintaining instruction-following capabilities.

# Conclusion:

The paper provides valuable insights into the impact of KV cache compression on LLMs' core capabilities.

It highlights the need for task-adaptive compression strategies and suggests that multi-step reasoning models are more robust to compression.

The proposed ShotKV method demonstrates significant improvements in performance, particularly for long-context generation tasks, making it a promising approach for efficient LLM deployment in resource-constrained environments.

# Impact Statement:

The research has positive implications for reducing computational resources and energy consumption, making AI technology more accessible.

However, it also raises concerns about the potential acceleration of LLM adoption and its broader societal impacts, such as employment and privacy. The authors emphasize the importance of responsible deployment and ethical considerations in the application of these technologies.

# 9. [1 2025] MUSTAFAR: Promoting Unstructured Sparsity for KV Cache Pruning in LLM Inference

We conduct a systematic exploration of pruning strategies and find per-token magnitude-based pruning as highly effective for both Key and Value caches under unstructured sparsity, surpassing prior structured pruning schemes.

The Key cache benefits from prominent outlier elements, while the Value cache surprisingly benefits from a simple magnitude-based pruning despite its uniform distribution. KV cache size is the major bottleneck in decode performance due to high memory overhead for large context lengths.

# 11. [106 2025] LightThinker：Thinking Step-by-Step Compression & 12. LightThinker2 From Reasoning Compression to Memory Management

# LightThinker vs LightThinker++: Comparative Analysis

# 1. Core Motivation

LightThinker (2025)

  • Focus: Efficient reasoning in LLMs by compressing intermediate thoughts (CoT reasoning steps) into concise representations, reducing memory and inference time.
  • Problem: Standard CoT produces large token sequences; KV cache size and attention computation grow with context length.
  • Key Idea: Introduce implicit thought compression via gist tokens.
  • Scope: Short- to medium-horizon reasoning tasks.

LightThinker++ (2026)

  • Focus: Extend LightThinker to handle complex, long-horizon reasoning with explicit adaptive memory management.
  • Problem: Pure compression risks irreversible loss of critical details.
  • Key Idea: Memory primitives (commit, expand, fold) for reversible archival and retrieval of past reasoning.
  • Scope: Standard reasoning and long-horizon agentic tasks.

Comparison: LightThinker++ moves from representation-level compression (implicit) to explicit behavioral-level memory management.

# 2. Methodology

# 2.1 Thought Compression

Feature LightThinker LightThinker++
Type Implicit, representation-level Explicit, behavioral-level
Mechanism Compresses Thought_i → Gist tokens (C_i) Memory primitives control visibility of each reasoning step
Attention Restricted to question + compressed tokens Dynamic projection based on archive/active state
Loss Risk Possible irreversible info loss Expand allows restoration of details

# 2.2 Memory Management

  • LightThinker++ introduces memory entity: ℐ_k = (R_k, Z_k)

    • R_k = raw reasoning
    • Z_k = semantic summary

  • Managed context ℋ̃_t = {m^(t)_1, ..., m^(t)_K} dynamically projects steps based on visibility σ_k ∈ {archive, active}.

  • Enables controlled compression, restoration, and high-fidelity reasoning.

# 3. Efficiency & Compression Gains

Metric LightThinker LightThinker++
Peak token reduction ~70% ~70% (Throughput), 45% (Budget)
Inference speed ~26% faster Slight overhead for expand, maintains high throughput
Dependency reduction ~78% 69.9% (Throughput), 33.7% (Budget)
Compression ratio ~4.5× 8–15× depending on task density
  • LightThinker++ adapts compression frequency to task difficulty.
  • Decouples reasoning depth from memory limits.

# 4. Accuracy & Robustness

  • LightThinker: Good accuracy; errors when compressed steps drop critical info.
  • LightThinker++: Expand/fold cycles restore lost info; improves accuracy on multi-step tasks.
  • Structured synthesized traces in LightThinker++ ensure high information density.

# 5. Training & Implementation

Aspect LightThinker LightThinker++
Training SFT with attention masks for compression SFT with memory-labeled trajectories, commit/expand/fold signals
Sequence length 10k tokens max 16k–32k, up to 110k for long-horizon tasks
GPUs 4×A800 8×A800
Data CoT & o1-like thinking traces Multi-source agentic + DeepResearch trajectories

# 6. Practical Implications

  1. KV Cache:

    • LightThinker: Compressed hidden states.
    • LightThinker++: Semantic summaries + selective raw retrieval.

  2. Inference Bottlenecks:

    • LightThinker: Faster due to fewer tokens, but lossy.
    • LightThinker++: Slight overhead for expand/fold, but long-horizon reasoning maintained.

  3. Task Suitability:

    • LightThinker: Standard reasoning tasks.
    • LightThinker++: Long-horizon, multi-round, high-density reasoning tasks.

# 7. Key Takeaways

  • From LightThinker to LightThinker++:

    1. Implicit → explicit memory control
    2. Single-level compression → hierarchical adaptive memory
    3. Short CoT → scalable to multi-round reasoning
    4. Maintains logical fidelity
    5. Aligns reasoning steps with memory lifecycle
Feature LightThinker LightThinker++
Compression Implicit gist tokens Explicit commit/expand/fold
Fidelity Some info loss Recoverable via expand
Memory Footprint Reduced, fixed Reduced, adaptive
Inference Speed Faster Slight overhead, better throughput under budget
Long-horizon tasks Limited Multi-step agentic reasoning
Training SFT with compression masks SFT with memory-labeled trajectories
KV Cache Compressed hidden states Semantic summaries + selective raw retrieval
Robustness Errors in dense logic High-fidelity maintained

# Conclusion

  • LightThinker: Efficient CoT reasoning with implicit compression.
  • LightThinker++: Scalable, accurate reasoning across standard and long-horizon tasks with explicit memory management.
  • Key advancement: Behavioral-level memory control and dynamic retrieval enable high-fidelity, efficient reasoning.
