LLM Midtraining

# 1.[66 2025] OctoThinker Mid-training Incentivizes Reinforcement Learning Scaling 2.[5 2025] A Survey on LLM Mid-training 3. [20 2025] Mid-Training of Large Language Models: A Survey

# 1.[66 2025] OctoThinker Mid-training Incentivizes Reinforcement Learning Scaling

# 1. Problem Addressed

Large language models (LLMs) behave differently during RL:

  • Qwen scales well,
  • Llama struggles with repetitive, unstable outputs.

Mid-training (between pre-training and RL) is identified as a critical lever to improve RL compatibility.

# 2. Key Insights

  • High-quality, reasoning-intensive corpora matter
    • MegaMath-Web-Pro and MegaMath-Web-Pro-Max significantly improve both base model performance and downstream RL outcomes.
    • Low-quality corpora (FineMath-4plus) cause repetitive, excessively long responses. evidenced by sudden performance drops and sharp increases in response length.
  • QA / Chain-of-Thought (CoT) data improves reasoning
    • Short-CoT has marginal RL gains.
    • Long-CoT substantially boosts RL performance but can destabilize training (response length spikes, sudden drops in accuracy).
  • Instruction-Following Data stabilizes RL and unlocks QA potential after RL training, incorporating instruction-following data, unlocks the potential of short-CoT data, showing performance advantages over the exclusion case after 200 steps. Additionally, this inclusion helps stabilize response length, resulting in smoother increases compared to when instruction-following data is excluded. But for training with long-CoT mid-training, Instruction-Following data fails to prevent the overall decline in RL performance and the rapid increase in response length.
    • Small amounts (~1%) help mitigate training collapse caused by long-CoT.
    • Improves formatting, response consistency, and overall RL effectiveness.
  • Progressive maximum response length scheduling stabilizes RL we start with a limit of 2,048 tokens for the first 200 steps, increase it to 4,096 tokens from step 200 to step 320, and then further expand to the full 8,192-token context length from step 320 to step 400
    • Gradually increasing max token length prevents spikes in response length and improves training smoothness.
  • Scaling mid-training budget consistently improves RL performance
    • Larger mid-training token counts (20B → 70B → 100B) boost RL outcomes, even if base model metrics show minor changes.

# 3. OctoThinker Approach

  • Two-stage mid-training:
    • Stable stage: Build reasoning foundations on high-quality math corpora (200B tokens).
    • Decay stage: Branch into long-CoT, short-CoT, and hybrid variants with learning rate decay (20B tokens).
  • RL stage: Each branch fine-tuned with reinforcement learning (GRPO) to produce models with diverse, stable reasoning behaviors.

# 2.[5 2025] A Survey on LLM Mid-training

# 1. Core Idea

The paper introduces mid-training as a distinct and crucial stage in the lifecycle of large language model (LLM) development. It sits between pre-training (foundation learning) and post-training (task-specific fine-tuning like RLHF or SFT). The central goal of mid-training is to enhance specialized capabilities—such as mathematics, reasoning, coding, and long-context understanding—while preserving the general knowledge and language skills learned during pre-training.

Key points:

  • Pre-training: Establishes general capabilities using broad corpora.
  • Mid-training: Bridges the gap, efficiently enhancing target skills using curated, often synthetic, datasets and tailored training strategies.
  • Post-training: Focused task-specific fine-tuning (e.g., instruction following, RLHF).

The survey formalizes mid-training, analyzes optimization frameworks, and reviews mainstream implementations, providing a taxonomy and actionable guidance for researchers.

# 2. Data Curation in Mid-Training

Mid-training datasets are a hybrid of general and specialized data, carefully constructed for specific objectives:

  • Collection: Sources include web crawls (e.g., CommonCrawl, Pile), books, QA pairs (Stack Exchange, RedStone), code repositories, and instruction datasets.
  • Synthesis: LLMs generate synthetic data to overcome scarcity:
  • Distillation: Generate high-quality data or teach smaller models.
  • Extraction: Pull structured QA or instruction data from existing corpora.
  • Evolution: Iteratively create enhanced problems and solutions (e.g., mathematical problems).
  • Selection: High-quality samples are chosen using:
  • Targeted sampling: Upsample specialized domains.
  • Rater-based filtering: Use scoring models like QuRater or FineWeb-Edu.
  • Decontamination: Remove sensitive or benchmark-overlapping data via n-grams, embeddings, or hybrid methods.
  • Mixture: Combine general corpora with specialized QA, instruction, and long-context data, balancing proportions for optimal performance (often 70% general, 30% specialized).

# 3. Training Strategies

Training strategies for mid-training focus on learning rate scheduling and other hyperparameters:

Learning Rate (LR) Scheduling:

  • Warm-up phase: gradually ramp up LR.
  • Decay phase: linear, cosine, exponential, or multi-stage WSD (Warmup-Stable-Decay) schedulers.
  • Rapid decay often paired with high-quality data for optimal convergence.
  • Other hyperparameters: Batch size adjustments affect stability and learning efficiency.

These strategies allow mid-training to achieve steep performance gains with less computation than pre-training.

# 4. Model Architecture Optimization

Key optimizations during mid-training include:

  • Long Context Extension: Techniques to handle sequences longer than the pre-training context limit:
    • RoPE (Rotary Position Embedding) variants:
    • PI (Position Interpolation)
    • NTK-aware interpolation
    • YaRN (hybrid approach)
  • Progressive, multi-stage extension with careful data composition (natural + synthetic).
  • Language Extension: Increase multilingual support, especially low-resource languages, by expanding vocabulary and introducing diverse multilingual datasets.

# 5. Objective-Driven Implementations

Mid-training objectives are categorized into four main domains:

  • General Capabilities: Preserve and refine general language and knowledge skills.
  • Core Cognitive Abilities: Knowledge mastery and reasoning improvement, particularly in math and STEM domains.
  • Task Execution Capabilities: Warm-up for post-training, covering:
  • Instruction following
    • Coding (generation, completion, fixing, reasoning)
    • Agentic tool usage (synthetic datasets for tool interactions)
  • Extension Capabilities:
    • Long-context reasoning
    • Multilingual capability enhancement

Empirical examples show that even a small proportion of specialized data in mid-training can significantly improve downstream performance

# 3. [20 2025] Mid-Training of Large Language Models: A Survey

# 1. Overview of Mid-Training

Mid-training is an intermediate stage between large-scale pre-training and task-specific supervised fine-tuning (SFT). Its purpose is to:

  • Refine data quality by emphasizing high-utility, low-noise data.
  • Adjust optimization schedules, particularly learning rates.
  • Extend context length, allowing models to handle longer sequences.

The theoretical rationale combines gradient noise scale, information bottleneck, and curriculum learning principles, promoting generalization and abstraction in the model instead of mere memorization.

# 2. Theoretical Foundations

  • Gradient Noise Scale (GNS)
    • Measures the amount of useful signal per gradient update.
    • High-quality data increases GNS, helping models escape sharp minima and avoid overfitting to noisy tokens.
  • Information Bottleneck
    • Represents a compression of internal features, retaining only task-relevant information.
    • Mid-training focuses on high-quality data that enhances structured reasoning and factual representations.
  • Curriculum Learning
    • Transition from general, diverse, and noisy data to more challenging, informative examples.
    • Reinforces complex reasoning skills such as chain-of-thought (CoT), math problem solving, and multi-step planning.

# 3. Data in Mid-Training

  • Mid-training prioritizes quality over scale, using several categories of data:

    • High-Quality Web Data: Curated, low-toxicity, diverse sources like CommonCrawl, Dolma, FineWeb-Edu.
    • Code and Math: Enhances symbolic reasoning and program synthesis capabilities (e.g., Stack v1/v2, StackMathQA, FineMath4+).
    • Instruction and QA-style Data: Improves alignment, reasoning, and instruction-following skills (e.g., Ultrachat, EvolInstruct, OpenOrca).
    • Synthetic Textbooks/Knowledge-Dense Data: Generated using LLMs or knowledge graphs to boost reasoning, STEM, and multilingual skills.
    • Long-Context Data: Documents spanning thousands of tokens to enhance memory, coherence, and long-range reasoning.
    • Reasoning and CoT Data: Step-by-step problem decomposition for logical reasoning.
    • Fill-in-Middle (FIM) Data: Supports bi-directional reasoning and better long-range dependency modeling.

  • Insights from data selection:

    • Even a small fraction of domain-specific data (math, code) can produce disproportionate gains.
    • Synthetic data amplifies learning efficiency, filling gaps in underrepresented domains.
    • Careful data repetition and rewriting can substitute for large-scale corpus expansion.
    • Maintaining distributional continuity prevents catastrophic forgetting.

# 4. Learning Rate Scheduling

Mid-training often uses annealing-based LR strategies:

  • Linear Warmup → Cosine/Linear Decay: Most common and empirically stable.
  • WSD (Warmup-Stable-Decay): Three-phase schedule enabling rapid convergence in decay stage, effective for high-quality mid-training data.
  • Power Scheduler: Generalizes WSD using a power-law relation for token/batch-size agnostic LR.
  • Multi-step Scheduler: Step-wise decay aligning with staged training (used in DeepSeek, MiniCPM).

Key insights:

  • LR scheduling strongly affects optimization stability and generalization.
  • Warm-up helps stabilize early training, enabling higher peak LR.
  • Decay strategies are system-dependent and interact with batch size, model scale, and optimizer choice.

# 5. Long-Context Extension

Mid-training often extends sequence length beyond typical 4K–8K limits:

Techniques:

  • Position Interpolation (PI) – scales all RoPE frequencies uniformly.
  • NTK-aware / NTK-by-parts – preserves high-frequency components for local fidelity.
  • YaRN – adds temperature scaling for stability at large contexts.
  • LongRoPE – learns token- and dimension-specific frequency scaling for ultra-long contexts.

Practical insights:

  • Frequency remapping + optional temperature scaling is a unified framework.
  • Architecture assists like interleaved local/global attention, MoE, GQA/MLA mainly reduce KV memory cost.
  • Progressive training stages (e.g., 4K → 32K → 128K) with mixed long/short tokens and synthetic tasks yield stable long-context performance
