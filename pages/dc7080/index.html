# 1. [55 2025] Scaling Long-Horizon LLM Agent via Context-Folding

The idea is simple.

For subtasks, make a branch off to another agent or the same agent, so it will conduct the task. When the task is done, return the result and fill into the context of the main task.

One of the fancy thing would be for the branches, they can share some of the KV cache, since they are originated from the same main task.

Takeaway: Use branch action and return action to manage the subtask. Core design in the reward function:

  • If the context in the main task is too long, then we can set penality to each token, so it will punish the main task from not generate new subtask.
  • If the subtask task drift from the original task, first we can use another model to judge whether this is a drift, if there is, we will punish the substask from being out of scope.

# FoldGRPO Reward and Penalty Design (Generated from Dr. Chatgpt)

# 1. Overall RL Objective

  • FoldGRPO optimizes the agent's trajectory, including both the main thread and branch sub-tasks.
  • The agent policy is conditioned on folded context (F(\tau_{<i})), not the full interaction history.
  • The learning objective combines:
    • Task success reward (R_i \in {0,1}) (pass/fail for the overall task)
    • Token-level process rewards (Q_{i,t}) that shape intermediate behavior.

Token-level Advantage Calculation:

A_{b_{i,t}} = clip(R_i + Q_{i,t}, 0, 1) - mean(R_i) / std(R_i)
1

Where (Q_{i,t}) regularizes branch-return behavior at the token level.

# 2. Token-Level Process Rewards and Penalties

# (a) Unfolded Token Penalty

  • Purpose: Prevents token-heavy operations in the main thread, which quickly exhausts context.
  • Implementation:
    • If the main context >50% of its limit, set (Q_{i,t} = -1) for all main thread tokens except branch-creating tokens.
  • Effect: Encourages moving heavy computations into branches.

# (b) Out-of-Scope Penalty

  • Purpose: Keeps branch actions focused on the assigned sub-task.
  • Implementation:
    • Use GPT-5-nano (This is interesting, like a judge model) to assess if branch actions deviate from the sub-task prompt.
    • If deviation detected, set (Q_{i,t} = -0.2) for all tokens in the branch.
  • Effect: Maintains scope integrity of each branch.

# (c) Failure Penalty

  • Purpose: Penalizes failed tool calls (e.g., unsuccessful searches or code execution).
  • Implementation:
    • Apply (Q_{i,t} = -1) for tokens in a failed tool call turn.
    • Other tokens get (Q_{i,t} = 0).
  • Effect: Encourages successful and careful tool usage.

# 3. Combined Effect

  • Main Thread: Learns to plan high-level strategies without clutter.
  • Branches: Learn to complete sub-tasks effectively and produce concise summaries.
  • Dense Feedback: Reduces the problem of sparse rewards from only task-level success.
  • Outcomes:
    • Main context compressed to ~8K tokens
    • 5–6 focused branches per task
    • Higher task success (e.g., pass@1 62% on BrowseComp-Plus)

# 4. Intuition

  • Think of the reward system as a real-time teacher:
    • "Don't overload main context → -1"
    • "Stay on your assigned sub-task → -0.2"
    • "Failed tool call → -1"
  • This shapes the agent to naturally follow the folding workflow without heuristics.

Reference: Weiwei Sun et al., "Scaling Long-Horizon LLM Agent via Context-Folding," 2025【6†source】.

# 2. [25 2025] Acon Optimizing Context Compression for Long-Horizon LLM Agents

Acon will use compressed failed and succeed and full context to refine the prompt. And then the prompt and the large compressor model could be distilled into a small model.

# Key Components

# 1. History & Observation Compression

  • History Compression: Compress past actions and observations when they exceed a threshold.
  • Observation Compression: Compress the current environment observation if it is long.
  • Compressed outputs replace the original sequences in the agent's input.

# 2. Compression Guideline Optimization

  • Optimize natural language prompts (compression guidelines) instead of LLM weights.

  • Contrastive feedback: Compare failed compressed trajectories with successful uncompressed trajectories.

  • Two-step iterative optimization:

    • Utility Maximization (UT): maximize task success.
    • Compression Maximization (CO): minimize context length while preserving task-critical information.

  • Fully gradient-free, works with closed-source or proprietary LLMs.

# 3. Distillation into Smaller Models

  • Compressors are distilled into smaller LMs to reduce computational overhead.
  • Maintains >95% of teacher performance while lowering inference cost.
  • Enables local deployment with smaller models for efficiency.

# 4. Adaptive & Model-Agnostic

  • Works with any LLM, including API-based and open-source models.
  • Supports multi-step, long-horizon tasks across diverse environments.

# Key Findings

  1. Memory Efficiency: Peak tokens reduced by 26–54% while largely preserving performance.
  2. Task Performance: Large LLMs maintain task accuracy; smaller agents improve by 20–46%.
  3. Distillation Success: Small distilled compressors retain over 95% of the teacher model's performance.
  4. Effectiveness for Small Agents: Smaller LLM agents can approach large-model performance using concise, informative contexts.
  5. Compression Clarifies Dependencies: In case studies, compression not only reduces size but also highlights critical dependencies, converting failed trajectories into successful ones.
  6. Threshold Tuning Matters: Moderate thresholds for history (4096 tokens) and observation (1024 tokens) provide the best trade-off between accuracy and efficiency.

*This summary is based on the paper ACON: Optimizing Context Compression for Long-Horizon LLM Agents (Minki Kang)
